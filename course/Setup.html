Introduction

Aims

The rest of this course is a read-only experience until you can actually compile and run a COBOL program on your own machine. This unit walks you through choosing a compiler, installing it, configuring an editor, and compiling a short Hello, World program end-to-end.

The course standardises on GnuCOBOL, an open-source COBOL compiler that runs on Windows, macOS and Linux. Every code example later in the course is intended to compile under GnuCOBOL without modification.

Objectives

By the end of this unit you should be able to -

  1. Explain why GnuCOBOL is a good default choice for learning COBOL.

  2. Install GnuCOBOL on your operating system and verify the install from the command line.

  3. Configure Visual Studio Code so COBOL source files have syntax highlighting and basic editor assistance.

  4. Compile and run a short COBOL program with the cobc compiler driver.

  5. Recognise and recover from the most common first-run errors.

Prerequisites

  • A computer running Windows 10/11, macOS or a recent Linux distribution.
  • Administrator (or sudo) access so a package manager can install GnuCOBOL.
  • Basic familiarity with opening a terminal (PowerShell, Terminal.app or a Linux shell) and running commands.

Pick a compiler

The current landscape

COBOL is an ISO standard, but there is no single canonical compiler. The three options a learner is most likely to encounter today are summarised below.

Compiler Cost Runs on Best for
GnuCOBOL Free / open source Windows, macOS, Linux, BSD Learning COBOL, running this course's examples locally.
IBM Enterprise COBOL Commercial z/OS mainframe only Production mainframe work. Out of reach without access to an IBM system or a hosted z/OS sandbox.
TutorialsPoint online compiler Free, browser-based Anywhere with a browser Trying a single snippet without installing anything. Not suitable for the file-handling chapters later in the course.

GnuCOBOL is the right default for this course. It is free, runs everywhere, and supports enough of the COBOL 85 / 2002 / 2014 standards to handle every example you will see in later units, including the sequential and indexed file chapters that the online compilers cannot run.

Install GnuCOBOL

Windows

The recommended path on Windows is MSYS2, a Unix-like environment that ships precompiled GnuCOBOL packages. Native Windows binaries also exist (e.g. arnoldtrembley's builds), but MSYS2 keeps the toolchain — compiler, headers, libraries — coherent in one place.

If you already use the Windows Subsystem for Linux, follow the Linux instructions inside your WSL distribution instead.

1. Install MSYS2

Download the MSYS2 installer from https://www.msys2.org/ and run it. Accept the default install location.

2. Install GnuCOBOL inside the MSYS2 shell

Open the MSYS2 MINGW64 shell from the Start menu and run:

pacman -Syu
pacman -S mingw-w64-x86_64-gnucobol

3. Verify the install

Still inside the MINGW64 shell, run:

cobc --version

You should see a line beginning with cobc (GnuCOBOL) followed by a version number.

macOS

Homebrew is the easiest way to install GnuCOBOL on macOS. The formula is named gnu-cobol.

1. Install Homebrew (if you do not already have it)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

2. Install GnuCOBOL

brew install gnu-cobol

3. Verify the install

cobc --version

Linux

Every mainstream distribution packages GnuCOBOL under either the gnucobol or open-cobol name. The two commands below cover Debian/Ubuntu and Fedora/RHEL-derived systems.

Debian / Ubuntu

sudo apt update
sudo apt install gnucobol

Fedora / RHEL / Rocky / Alma

sudo dnf install gnucobol

Verify the install

cobc --version

Pick an editor

Visual Studio Code

Any plain-text editor will compile COBOL — including Notepad, vim or nano — but Visual Studio Code with a COBOL extension gives you syntax highlighting, IntelliSense (autocomplete) and column-aware formatting, all of which materially shorten the feedback loop while you are learning.

  1. Install Visual Studio Code from https://code.visualstudio.com/.

  2. Open the Extensions panel (Ctrl+Shift+X on Windows and Linux, Cmd+Shift+X on macOS).

  3. Search for COBOL and install the extension by bitlang (its identifier is bitlang.cobol). This is the most widely used community extension and works without needing a paid licence.

  4. Open a folder, create a file ending in .cob or .cbl, and VS Code will activate the COBOL language mode automatically. You should see keywords like IDENTIFICATION DIVISION render in colour.

If you prefer a different editor, both Vim (cobol.vim) and Emacs (cobol-mode) have community-maintained modes that provide at minimum syntax highlighting.

Compile and run Hello, World

The program

The directive >>SOURCE FORMAT IS FREE tells GnuCOBOL to use the modern free-format source layout instead of the historical fixed columns. The rest of the course uses the same convention.

Save the listing below into a file called hello.cob:

       >>SOURCE FORMAT IS FREE
IDENTIFICATION DIVISION.
PROGRAM-ID. Hello.

PROCEDURE DIVISION.
    DISPLAY "Hello, World!"
    STOP RUN.

Compile

The -x flag tells cobc to build a standalone executable. Without it, cobc defaults to producing a callable module instead.

From the same directory as hello.cob, run:

cobc -x hello.cob

If the compile succeeds you will see no output at all. cobc will produce an executable named hello (or hello.exe on Windows) next to the source file.

Run

On macOS and Linux:

./hello

On Windows (MSYS2 MINGW64 shell):

./hello.exe

If that line appears, your toolchain is working end-to-end and you are ready to move on.

What can go wrong

Common first-run errors

The five items below cover roughly all of the failures a learner will hit in the first hour. Skim them now so the error messages look familiar when you see them.

cobc: command not found

Your shell cannot find the GnuCOBOL compiler. On Windows, make sure you are running the MSYS2 MINGW64 shell rather than a regular Command Prompt or PowerShell window. On macOS, run brew doctor to check that the Homebrew binary directory is on your PATH. On Linux, confirm the package is installed with which cobc.

error: PROGRAM-ID header missing or unexpected IDENTIFICATION

Almost always a source-format mismatch. GnuCOBOL defaults to fixed format, where columns 1-6 are a sequence area, column 7 is an indicator, area A runs columns 8-11 and area B runs columns 12-72. If you typed the program flush against the left margin and did not include >>SOURCE FORMAT IS FREE at the top, the compiler will read your code as sequence numbers and reject it. Either add the directive (recommended) or pass -free on the command line: 

cobc -x -free hello.cob
Code past column 72 in fixed format

In fixed format, anything in columns 73 and beyond is silently ignored by the compiler — it is the historical card-image identification area. If a statement in fixed format appears to be missing its tail (for example a DISPLAY stops mid-string), check that you have not wrapped past column 72.

Wrong file extension

GnuCOBOL recognises .cob, .cbl and .cpy (for COPY library files) by default. A file called hello.txt will compile only if you explicitly tell cobc the source format. Stick to .cob while learning.

libcob.so: cannot open shared object file at run time

On Linux, the GnuCOBOL runtime library was found at compile time but not at run time. This usually means the binary built with one GnuCOBOL package and you are running it from an environment that no longer sees the runtime — for example running from a different shell after a partial uninstall. Re-running sudo apt install gnucobol (or your distribution's equivalent) restores it.
